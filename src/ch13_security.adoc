== Bitcoin 安全

保护你的比特币是一项挑战，因为比特币不像银行账户里的余额。比特币更像是数字现金或黄金。你可能听过这样一句话："占有即九成的法理。"在比特币世界里，占有就是十成的法理。掌握能花费某些比特币的密钥，就等同于持有现金或一块贵金属。你可能会丢失它、放错地方、被人偷走，或者不小心把错误的金额发给了别人。在上述任何一种情形下，用户都无法借助协议本身来追回损失，就像你把现金掉在公共人行道上一样。

不过，比特币系统也具备现金、黄金和银行账户所没有的能力。装着你的密钥的 Bitcoin 钱包可以像任何文件一样进行备份，可以保存多份副本，甚至可以打印在纸上作为硬拷贝备份。现金、黄金或银行账户都无法"备份"。比特币和此前出现过的任何事物都有着足够大的差别，因此我们也需要用一种全新的方式来思考如何保护自己的比特币。

=== 安全原则

((("Bitcoin", "security", "principles of", id="bitcoin-security-principle")))((("security", "principles of", id="security-principle")))((("decentralized consensus", "as security principle", secondary-sortas="security principle", id="decentral-consensus-principle")))比特币的核心原则是去中心化，它对安全性有着重要影响。像传统银行或支付网络这样的中心化模型，依赖访问控制和资质审查来把恶意行为者挡在系统之外。相比之下，像 Bitcoin 这样的去中心化系统则把责任与控制权交还给用户。由于网络安全建立在独立验证之上，网络可以是开放的，Bitcoin 流量也不需要加密（尽管加密仍然可能有用）。

在传统支付网络（例如信用卡系统）中，支付是开放式的，因为它包含了用户的私有标识（信用卡号）。在首次扣款之后，任何拿到这个标识的人都可以"拉取"资金，对持卡人反复扣款。因此，支付网络必须端到端加密，并确保没有窃听者或中间人能够在传输途中或存储状态（静态）下破坏支付流量。一旦恶意行为者侵入了系统，他就可以破坏当前的交易，_并_ 拿到可以用来发起新交易的支付凭证。更糟的是，当客户数据被泄露后，客户面临身份被盗用的风险，必须采取行动防止被泄露的账户被欺诈性使用。

Bitcoin 则截然不同。一笔 Bitcoin 交易只授权将特定金额转给特定收款人，且无法被伪造。它不会泄露任何隐私信息（例如交易双方的身份），也无法被用来授权额外的支付。因此 Bitcoin 支付网络不需要加密，也不需要防止窃听。事实上，你完全可以通过开放的公共信道（例如未加密的 WiFi 或蓝牙）广播 Bitcoin 交易，而不会损失任何安全性。

Bitcoin 的去中心化安全模型把大量的权力交到用户手中。伴随这种权力的，是用户必须自己保管好密钥的责任。对大多数用户来说这并不容易，尤其是在通用计算设备上（比如联网的智能手机或笔记本电脑）。虽然 Bitcoin 的去中心化模型可以避免信用卡那种大规模泄露的情形，但许多用户依然无法妥善地保护自己的密钥，被一个一个地黑掉。

==== 安全地开发 Bitcoin 系统

对 Bitcoin 开发者而言，一个关键原则就是去中心化。大多数开发者熟悉的是中心化安全模型，可能会很自然地想把这些模型套用到自己的 Bitcoin 应用上，结果往往是灾难性的。

Bitcoin 的安全性依赖于对密钥的去中心化控制，以及用户对交易的独立验证。如果你想充分利用 Bitcoin 的安全性，就必须确保自己仍然处于 Bitcoin 的安全模型之内。简单来说：不要从用户手中夺走对密钥的控制权，也不要把验证外包出去。

举个例子，许多早期的 Bitcoin 交易所把所有用户资金都集中在一个"热"钱包里，密钥存放在单一服务器上。这样的设计把控制权从用户手中拿走，又把对密钥的控制权集中到一个系统中。很多这样的系统都被黑过，给客户造成了灾难性的后果。

除非你准备像传统银行那样在运营安全、多层访问控制和审计上投入巨大资源，否则在把资金移出 Bitcoin 去中心化安全语境之前，应该非常谨慎地考虑。即便你有足够的资金和纪律来实施一套稳健的安全模型，这种设计也只是在复制传统金融网络那种脆弱的模型——它们正深受身份盗用、腐败和挪用之苦。要利用好 Bitcoin 独特的去中心化安全模型，你必须抵御中心化架构的诱惑——它们也许让人感到熟悉，但最终会颠覆 Bitcoin 的((("decentralized consensus", "as security principle", secondary-sortas="security principle", startref="decentral-consensus-principle")))安全性。

==== 信任之根

传统((("root of trust", id="root-trust")))安全架构建立在一个被称为 _信任之根（root of trust）_ 的概念之上：一个受信任的核心，作为整个系统或应用安全性的基础。安全架构以信任之根为中心，像洋葱的层一样向外形成一圈圈同心圆，把信任从中心向外延伸。每一层都借助访问控制、数字签名、加密和其他安全原语，构建在更受信任的内层之上。随着软件系统变得越来越复杂，它们越可能含有 bug，从而更容易遭到安全攻陷。因此，软件系统越复杂，就越难以做到安全。信任之根的概念确保系统中绝大部分信任都被放在最不复杂、因而也最不易受攻击的部分，而更复杂的软件则一层一层地围绕着它构建。这种安全架构在不同尺度上反复出现：先在单一系统的硬件中建立信任之根，再通过操作系统把这个根延伸到更高层次的系统服务，最终延伸到许多服务器，形成一圈圈信任程度逐渐降低的同心圆。

Bitcoin 的安全架构有所不同。在 Bitcoin 中，共识系统构建出一条完全去中心化的、可信的区块链。一条经过正确验证的区块链以创世区块作为信任之根，从那里出发逐步构建出一条延伸至当前区块的信任链。Bitcoin 系统可以、也应当以区块链作为自己的信任之根。在设计一个由许多不同系统上的多个服务组成的复杂 Bitcoin 应用时，你应当仔细审视它的安全架构，弄清楚信任究竟被放在了哪里。归根结底，唯一应该被显式信任的，就是一条经过完整验证的区块链。如果你的应用显式或隐式地把信任寄托在区块链以外的任何东西上，那都应当是一个值得警惕的信号，因为这会引入脆弱性。评估你的应用安全架构的一个好方法是：逐一考察每个组件，并设想该组件被完全攻陷、落入恶意行为者控制下的情形。逐个分析你的应用中的每个组件，评估它被攻陷后对整体安全性的影响。如果在某个组件被攻陷之后你的应用就不再安全，那就说明你把信任错放在了那些组件上。一个没有漏洞的 Bitcoin 应用，应当只在 Bitcoin 共识机制本身被攻破时才会出问题——也就是说，它的信任之根建立在 Bitcoin 安全架构中最强的那一部分之上。

大量被黑的 Bitcoin 交易所的案例恰好佐证了这一点：它们的安全架构与设计经不起哪怕最粗略的审视。这些中心化的实现把信任显式地寄托在 Bitcoin 区块链之外的许多组件上，例如热钱包、中心化数据库、易受攻击的加密密钥，以及((("Bitcoin", "security", "principles of", startref="bitcoin-security-principle")))((("security", "principles of", startref="security-principle")))((("root of trust", startref="root-trust")))类似的方案。

=== 用户安全最佳实践

((("Bitcoin", "security", "best practices", id="bitcoin-security-best-practice")))((("security", "best practices", id="security-best-practice")))((("best practices, security", id="best-practice-security")))人类已经使用物理安全手段数千年了。相比之下，我们在数字安全方面的经验还不到 50 年。现代通用操作系统并不算很安全，也并不特别适合用来存放数字货币。我们的计算机通过常开的互联网连接，持续暴露在外部威胁之下。它们运行着来自上百名作者的数千个软件组件，这些组件通常对用户文件拥有不受限制的访问权。安装在你电脑上的成千上万个软件中，只要有一个流氓软件，就可能记录你的键盘输入、窃取你的文件，盗走钱包应用中存放的任何比特币。要把一台计算机维护得没有病毒、没有木马，所需的水平远超绝大多数普通计算机用户的能力范围。

尽管信息安全领域经过了数十年的研究与发展，数字资产对于有备而来的攻击者来说仍然脆弱得令人遗憾。即使是金融服务公司、情报机构和国防承包商中那些防护最严密、限制最多的系统，也屡屡被攻破。Bitcoin 创造了具有内在价值的数字资产，它们可以被瞬间且不可逆地盗走并转移到新的所有者手中。这对黑客构成了巨大的诱惑。在此之前，黑客攻陷系统后，必须先把身份信息或账户凭据（如信用卡和银行账户）变现。即便变现和洗钱过程困难重重，我们依然看到了不断升级的盗窃案。Bitcoin 把这一问题进一步放大，因为它根本不需要变现或洗钱——比特币本身就是有价值的。

Bitcoin 同时也为改善计算机安全提供了动力。过去，计算机被攻陷的风险既模糊又间接，而 Bitcoin 让这些风险变得清晰而显而易见。在计算机上持有比特币，会让用户切实意识到提升计算机安全的必要性。作为 Bitcoin 与其他数字货币普及和采用度上升的直接结果，我们看到了黑客手段与安全方案两方面同时升级。简单来说，黑客现在有了一块非常诱人的目标，而用户也有了明确的动力来保护自己。

过去三年里，作为 Bitcoin 普及的直接结果，我们在信息安全领域看到了大量创新，包括硬件加密、密钥存储和硬件签名设备、多重签名技术以及数字托管。在接下来的几节里，我们会探讨一些面向用户的实用安全最佳实践。

==== 比特币的实物存储

由于大多数((("bitcoins", "physical storage")))((("physical bitcoin storage")))((("storing bitcoins", id="storing-bitcoin")))用户对物理安全比对信息安全更得心应手，因此一种非常有效的保护比特币的方法，就是把它们转换成实物形式。Bitcoin 密钥以及用来生成它们的种子，本质上就是一串很长的数字。这意味着它们可以以实物形式保存，例如打印在纸上，或者蚀刻在金属板上。这样一来，保护密钥就简化成了对种子打印件进行物理保管。打印在纸上的种子被称作"纸质备份（paper backup）"，许多钱包都可以生成它。
让比特币离线的做法被((("cold storage")))称为 _冷存储（cold storage）_，这是最有效的安全手段之一。冷存储系统会在一台离线（从未联网）的设备上生成密钥，然后离线保存在纸上或 USB 存储棒等数字介质上。

==== 硬件签名设备

从((("hardware signing devices")))长期来看，Bitcoin 的安全会越来越多地以防篡改的硬件签名设备的形式呈现。与智能手机或台式电脑不同，Bitcoin 硬件签名设备只需要保存密钥，并使用它们来生成签名。由于没有可被攻陷的通用软件，加上接口非常有限，硬件签名设备能够为非专家用户提供强大的安全保障。硬件签名设备可能会成为存储比特币的主流方式。

==== 确保你能够访问

虽然大多数用户((("backing up", "importance of")))担心的都是比特币被盗，但实际上还有一个更大的风险。数据文件总是在丢失。如果它们里面装着 Bitcoin 密钥，那损失就会痛苦得多。在努力保护自己 Bitcoin 钱包的同时，用户必须非常小心，不要走得太远以至于最后把自己的比特币给弄丢了。2011 年 7 月，一个知名的 Bitcoin 宣传与教育项目损失了将近 7,000 枚比特币。为了防盗，所有者实施了一套复杂的加密备份方案。最终他们却意外丢失了加密密钥，让备份变得毫无用处，损失了一笔巨大的财富。这就像把钱埋在沙漠里藏起来一样：如果你把比特币保护得太好，可能就再也找不到它们了。

==== 分散风险

你会((("risk diversification")))((("storing bitcoins", startref="storing-bitcoin")))把全部身家以现金形式装在钱包里吗？大多数人都会觉得这样做太鲁莽，可 Bitcoin 用户却常常把全部比特币都放在同一个钱包应用里。相反，用户应当把风险分散到多个、不同的 Bitcoin 应用之间。谨慎的用户只会把一小部分（比如不到 5%）的比特币放在在线或移动钱包中作为"零钱"。其余部分应当拆分到几种不同的存储机制中，比如桌面钱包和离线（冷存储）。

==== 多重签名与治理

每当某家公司或个人((("multisignature addresses")))((("addresses", "multisignature")))保管大量比特币时，都应当考虑使用多重签名的 Bitcoin 地址。多重签名地址通过要求多于一把签名才能完成支付，从而保护资金。签名所用的密钥应当分别存放在多个不同地点，并由不同的人控制。例如在公司环境中，密钥应当独立生成，由多位公司高管分别持有，以确保任何单个人都无法独自挪用资金。多重签名地址也可以提供冗余：由一个人持有多把密钥，分别存放在不同的地点。


==== 可继承性

一个经常被忽视的((("survivability (of bitcoin access)")))((("estate planning")))重要安全考量是可用性，尤其是在密钥持有者丧失行为能力或去世的情境下。Bitcoin 用户被告知要使用复杂的密码，把密钥保管得严严实实、不与任何人分享。可惜的是，这种做法使得在用户无法亲自解锁时，他的家人几乎不可能找回任何资金。事实上，在多数情况下，Bitcoin 用户的家人可能完全不知道这些比特币资金的存在。

如果你持有大量比特币，应当考虑把访问细节分享给一位可信赖的亲属或律师。还可以借助多重签名访问，以及通过专门从事"数字资产遗嘱执行人"业务的律师所做的遗产规划，搭建出一套更复杂的可继承方案。

Bitcoin 是一项复杂的新技术，开发者们仍在不断探索它。随着时间推移，我们会逐步开发出更易于非专家使用的安全工具和实践方法。在此之前，Bitcoin 用户可以借助本章讨论的诸多建议，享受安全无忧的 Bitcoin ((("Bitcoin", "security", "best practices", startref="bitcoin-security-best-practice")))((("security", "best practices", startref="security-best-practice")))((("best practices, security", startref="best-practice-security")))体验。
